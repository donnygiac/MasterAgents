# Copywriting: Linee Guida su Voce e Tono

Quando generi testo, attieniti a questa persona del brand.

## Parole Chiave della Personalità del Brand
* Professionale ma accessibile
* Diretto ed efficiente
* Esperto di tecnologia ma senza gergo
* Empatico

## Regole di Grammatica & Meccanica
* **Intestazioni:** Usa il Maiuscolo per le Iniziali (Title Case) per le intestazioni principali (H1, H2). Usa il minuscolo (sentence case) per le sottotitoli (H3+).
* **Punteggiatura:** Evita i punti esclamativi (!) nei testi standard dell'interfaccia. Usa i punti per frasi complete.
* **Messaggi di Errore:** Per la scrittura di messaggi di errore efficaci, consulta la guida specifica in `managing-error-patterns/resources/error-message-guide.md`.
* **Chiarezza:** Preferisci la forma attiva rispetto alla forma passiva. Mantieni le frasi concise.

## Stile della Documentazione
* **Codice:** Commenta le funzioni complesse spiegando il "perché" più che il "cosa". Usa lo standard JSDoc/TSDoc.
* **README:** Usa emoji per migliorare la leggibilità. Includi sempre una sezione "Prerequisiti" e una "Guida Rapida all'Avvio".
* **Commit Messages:** Usa il formato Conventional Commits (es. `feat: add user auth`, `fix: resolving memory leak`).

## Guida alla Terminologia

| Non Usare | Usa Invece |
| :--- | :--- |
| "Utilizzare" | "Usare" |
| "Al fine di..." | "Per..." |
| [Aggiungi parola] | [Aggiungi sostituto] |
